The Project Card — Beacon's Truth Unit

Easy to maintain, strong enough to power both Q&A and dashboards.

Identity & Routing

Beacon ID, Title, Owner, Function/Domain, Stage, Truth State, Update Cadence

Intent

Problem statement, Scope, Target users, Expected outcomes (1-3 categories)

Evidence

Evidence links (one-pager, deck, demo), Data sources, Validation notes

Health, Risk, Mitigation

Delivery health (G/A/R + reason), Milestones (3-5), Risks with impact + mitigation

Investment & Resourcing

Budget band, Source split (DDIT/Function/Mixed), FTE band, Key roles

Links & Relationships

Tags/streams, Dependencies, Related initiatives (manual + auto-suggest)

Publication Gates — Trust by Design

Strict enough to preserve trust. Not so strict that nothing gets published.

Draft

Contributors only, not searchable

Requires: Identity + intent (owner, title, domain, problem statement)

→

Searchable

Default for innovators

Requires: Evidence link (1 min), Stage + Truth State, Update cadence

→

Leadership-Visible

Eligible for cockpit counts

Requires: Health (G/A/R + reason), Budget band + source, 1 milestone or 1 risk

Staleness rule: If last update > 90 days: flagged "Stale" and excluded from leadership "green" totals until refreshed.

Q&A Engine — A Responsible Librarian

1. User asks a question
↓
2. Retrieve from Project Card data + linked evidence
Structured search (SQL) + Semantic search (embeddings)
↓
3. Ground & Respond with inline citations
[Card: BCN-0042, Field: delivery_health]
↓
4. If missing: "Not available in Beacon yet.
The [field] would need to be filled by [owner role]."

Allowed (v1.0)

  • Summarize an initiative
  • Compare (structured fields)
  • List by filter
  • Status / health / budget band
  • Identify owners + evidence
  • Surface risks + mitigations

Disallowed (v1.0)

  • Guess budget beyond bands
  • Infer ownership if not stated
  • Claim strategy unless Committed
  • Recommend actions

Trust by Design

Beacon earns leadership trust through systematic constraints, not promises.

Grounding Rules

  • Every factual claim traceable to a card field or evidence doc
  • Inline citations on every Q&A answer
  • Confidence flags: grounded / partial / not available

Staleness Enforcement

  • 90-day staleness flag
  • Auto-downgrade from leadership cockpit
  • Excluded from "green" health totals

Publication Gates

  • 3-tier progressive requirements
  • Auto-validation before tier promotion
  • Domain curator approval for leadership visibility
Failure mode design: When data is missing, Beacon names the gap, names the likely owner, and suggests the fix. It never fabricates.
